## 二、Queue 模块 ([`queue.md`](../archive/queue.md))

### 核心实现

| 组件 | 说明 | 适用场景 |
|------|------|----------|
| **RingQueue** | 非线程安全环形队列 | 单线程内部使用 |
| **SpscQueue** | 单生产者单消费者无锁队列 | 多线程 RxQ/TxQ |
| **QueueConfig** | 队列配置结构 | 创建时指定行为 |

### 无锁设计特点

- 使用原子操作（AtomicUsize、AtomicBool）
- Acquire/Release 内存语义保证数据可见
- 位掩码取模：`index & mask` 代替 `index % capacity`

### 等待策略

```rust
pub enum WaitStrategy {
    Spin,                    // 自旋等待（CPU忙等待）
    Yield,                   // 休眠等待（yield让出CPU）
    Timeout(Duration),       // 超时后返回错误
}
```

### 队列常量

| 常量 | 默认值 |
|------|--------|
| DEFAULT_QUEUE_CAPACITY | 256 |
| MIN_QUEUE_CAPACITY | 2 |
| MAX_QUEUE_CAPACITY | 65536 |
| DEFAULT_SPIN_COUNT | 1000 |
| DEFAULT_TIMEOUT_MS | 100 |

## 三、Pool 模块 ([`pool.md`](../archive/pool.md))

### 模块结构

```
src/common/pool/
├── clear.rs          # Clear trait
├── config.rs         # AllocStrategy, WaitStrategy, PoolConfig
├── error.rs          # PoolError
├── stats.rs          # PoolStats, PoolStatus
├── pooled.rs         # Pooled<T> 包装器
├── pool.rs           # 通用对象池 Pool<T>
├── packet_pool.rs    # PacketPool, PooledPacket, PacketBuilder
└── mod.rs           # 模块导出
```

### 分配策略

| 策略 | 实现方式 | 适用场景 |
|------|----------|----------|
| Fifo | `idle.remove(0)` | 公平性要求高 |
| Lifo | `idle.pop()` | 低延迟场景 |
| Random | 简化为 Lifo | 高并发场景 |
| Recent | 从尾部弹出 | CPU 缓存友好 |

### 预设配置

| 配置 | 容量 | Packet大小 | 适用场景 |
|------|-------|-----------|----------|
| Default | 100 | 1514B | 通用场景 |
| high_throughput | 1000 | 9000B | 大流量传输 |
| low_latency | 500 | 1514B | 实时通信 |
| memory_constrained | 50 | 1514B | 嵌入式设备 |

### Pooled<T> 自动归还

```rust
pub struct Pooled<T> {
    inner: Option<T>,
    pool: Option<Arc<Pool<T>>>,
    released: AtomicBool,
}

impl<T> Drop for Pooled<T> {
    // 离开作用域时自动归还
}
```

## 九、设计原则总结

1. **零外部依赖** - 仅使用 Rust 标准库
2. **类型安全** - 充分利用 Rust 类型系统
3. **零拷贝** - 使用切片引用避免不必要的数据拷贝
4. **无锁并发** - 原子操作实现 SPSC 队列
5. **自动资源管理** - Drop trait 确保资源自动释放
6. **配置驱动** - 通过配置结构灵活控制行为
7. **中文文档** - 符合学习型项目定位
8. **完整测试** - 每个模块都有单元测试覆盖
