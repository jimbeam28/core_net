# CoreNet 网络协议栈架构设计

## 1. 项目概述

### 1.1 目标
设计并实现一个用于学习/研究的网络协议栈，支持完整的TCP/IP协议族。

### 1.2 设计目标
- **可学习性**: 代码结构清晰，便于理解网络协议原理
- **可扩展性**: 模块化设计，便于添加新协议支持
- **正确性**: 遵循RFC标准，实现正确的协议行为

## 2. 整体架构

### 2.1 线程模型（纯模拟模式）

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│  测试/模拟    │  ───>  │  处理线程    │  ───>  │  结果输出     │
│  注入器      │  RxQ   │ (Engine)     │  TxQ   │  (可选)       │
└──────────────┘         └──────────────┘         └──────────────┘
```

**说明**：
- **纯模拟环境**：不使用TUN/TAP等真实网络接口
- **RxQ**: 接收队列，用于注入测试报文
- **处理线程**: 从RxQ取报文，逐层解析，结果放入TxQ
- **TxQ**: 发送队列，用于收集处理结果
- **结果输出**: 可选的输出模块，打印或保存TxQ中的报文

### 2.2 模拟数据流

```
上行（解析）测试：
  测试报文 -> RxQ -> 处理线程解析 -> TxQ（结果）

下行（封装）测试：
  应用数据 -> 处理线程封装 -> TxQ（结果）
```

│                     应用层接口                          │

│                    协议处理引擎                         │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │ TCP     │  │ UDP     │  │ ICMP    │  │  ...    │ │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘ │
│       └─────────────┴─────────────┴─────────────┘        │
│  ┌─────────┐  ┌─────────┐                             │
│  │ IPv4    │  │ IPv6    │                             │

## 3. 报文处理流程

### 3.1 上行（解析）流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  测试注入    │ -> │  接收队列    │ -> │  处理线程    │ -> │  结果输出     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       |                  |                  |                  |
 1. 注入测试报文    2. 存入队列      3. 逐层解析:       6. 输出结果
                   (hex/bytes)       - 以太网层       (可打印/保存)
                                                - IP层
                                                - 传输层
       |                  |                  |                  |
                                                v
                                          ┌─────────────┐
                                          │  发送队列    │ <- 5. 需要响应
                                          │  (结果)     │
                                          └─────────────┘
```

### 3.2 下行（封装）流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  测试输入    │ -> │  处理线程    │ -> │  发送队列    │ -> │  结果验证     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       |                  |                  |                  |
 1. 提供发送数据    2. 逐层封装:      5. 存入队列      6. 验证结果
                   - 传输层         (可打印/保存)
                   - IP层
                   - 以太网层
       |                  |                  |                  |
```

### 3.3 Packet所有权流转

```
上行: 注入器创建Packet -> 移动到RxQ -> 处理线程消费解析 -> (可选)移动到TxQ

下行: 测试创建Packet -> 移动到处理线程封装 -> 移动到TxQ -> 输出/验证
```

## 4. 协议分层

| 层级 | 协议 | 职责 |
|------|------|------|
| 应用层 | Socket API | 提供网络编程接口 |
| 传输层 | TCP/UDP/ICMP | 端到端通信、错误报告 |
| 网络层 | IPv4/IPv6 | 路由、分片 |
| 链路层 | Ethernet/ARP | MAC寻址、地址解析 |

## 8. 技术选型

| 项目 | 选择 | 说明 |
|------|------|------|
| 运行模式 | 纯模拟 | 不使用真实网络接口，仅队列传递 |
| 队列实现 | 自定义环形缓冲区 | 学习数据结构原理 |
| 队列类型 | SPSC（单生产者单消费者） | 注入器/处理线程、处理线程/输出之间传递 |
| 错误处理 | 手动实现Error enum | 学习Rust错误处理 |
| 外部依赖 | 零依赖 | 纯标准库实现 |
| 并发模型 | 单线程处理 | 处理线程单线程逐层解析 |
| 测试方式 | 报文注入 | 从hex/文件注入测试报文 |
